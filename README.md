# 🔐 OWASP Top 10 — Interactive Study Guide

> **Break it to Defend it.**  
> A free, interactive study guide covering all 10 OWASP vulnerabilities — built for pentest interview prep, but explained so anyone can understand it.

🌐 **Live Site:** [muhammedp07.github.io/Owasp_top_10__study_guide](https://muhammedp07.github.io/Owasp_top_10__study_guide/)

## What's Inside

Each of the 10 vulnerabilities includes:

- 🍕 **Plain-English analogy** — explained so a non-technical person gets it
- 💻 **Real attack example** — with actual payloads and code
- ⚔️ **Attack vs. Defense breakdown** — what the attacker does, how defenders stop it
- 🎤 **Interview questions** — the ones actually asked at security firms
- ✅ **Mastery tracker** — mark each vuln as learned and track your progress

---

## Vulnerabilities Covered

| # | Vulnerability | Key Concepts |
|---|---------------|--------------|
| A01 | Broken Access Control | IDOR, privilege escalation, forced browsing |
| A02 | Cryptographic Failures | MD5, weak TLS, data exposure |
| A03 | Injection | SQLi, XSS, command injection |
| A04 | Insecure Design | Business logic flaws, threat modeling |
| A05 | Security Misconfiguration | Default creds, open S3, debug mode |
| A06 | Vulnerable & Outdated Components | Log4Shell, CVEs, supply chain |
| A07 | Identification & Auth Failures | Brute force, credential stuffing, session issues |
| A08 | Software & Data Integrity Failures | Insecure deserialization, SolarWinds, CI/CD |
| A09 | Security Logging & Monitoring Failures | SIEM, blind attacks, Equifax breach |
| A10 | Server-Side Request Forgery (SSRF) | Cloud metadata, internal recon, Capital One |

## Quick Recall Cheatsheet

```
A01 — Broken Access Control     → IDOR · priv esc · force browse
A02 — Crypto Failures           → MD5 · no TLS · data exposed
A03 — Injection                 → SQLi · XSS · CMDi
A04 — Insecure Design           → business logic · no threat model
A05 — Misconfiguration          → defaults · open S3 · debug on
A06 — Outdated Components       → Log4Shell · CVEs · no patching
A07 — Auth Failures             → weak session · brute · stuffing
A08 — Integrity Failures        → deserialize · SolarWinds · CI/CD
A09 — Logging Failures          → blind to attacks · no SIEM
A10 — SSRF                      → server as proxy · metadata API
```
